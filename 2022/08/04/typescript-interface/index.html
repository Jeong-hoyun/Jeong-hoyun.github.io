타입스크립트 interface

공통적으로 사용하는 부분을 지정하는 방식이 TypeScript type alias 방식과 다르다는 것을 배웠다. interface는 class 와 비슷한 방식으로 전의 데이터를 받아들이는 상속 방식이며 alias방식으로도 상속은 가능하지만 어떤 것이 더 좋을지는 아직 잘 모르겠다

interface의 강력한 성능 1 readonly

데이터 베이스에서 값을 가져오는 기본 키(primary key)는 절대로 변경되어서는 안되는 값이기에 readonly 를 통해서 값이 변경되지 않고 읽을수만 있도록 강제처리하는 기능이다. javascript에서는 해본적이 없는 방식이기에 아마 push 와 같은 방식을 막아주는 기능이 아닐까 생각해본다
